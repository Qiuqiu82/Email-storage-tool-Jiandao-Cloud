本次实现了什么功能

实现了邮件意图智能路由功能，系统能自动判断一封邮件是否属于出入库任务单，如果不是则跳过后续订单解析、策略校验和简道云写入流程，并向用户发送标准化提示回复。

通过什么方法实现

  1. 新增独立节点 02_邮件意图路由节点.py，专门负责从邮件主题、正文、附件名中提取关键词或解析大模型输出，归类为'出入库'或'其他'
  2. 在附件解析、策略校验、简道云写入三个已有节点中统一增加前置判断：若 mail_intent != '出入库' 或 is_inoutbound_mail == False，则直接跳过核心逻辑并填充空/默认结果
  3. 将邮件意图判断结果（mail_intent、is_inoutbound_mail）和固定提示语（fixed_reply_message）注入 variables，供下游节点读取
  4. 在简道云写入节点中根据邮件意图动态生成不同内容的自动回复消息，并结构化记录完整执行摘要
  5. 优化附件解析和策略校验中的第三方库导入方式，改为按需导入 requests 和 openpyxl，避免全局依赖冲突
  6. 重构 Excel 表格读取逻辑，用 openpyxl 替代 pandas，提升大文件解析稳定性与内存效率

功能执行流程

  1. 用户发送邮件 → 系统提取 email_info（含 subject、mail_body、attachments）→ 进入邮件意图路由节点
  2. 路由节点调用 _normalize_intent 尝试解析大模型返回的原始意图；失败则 fallback 到关键词匹配（如'入库''出库''订单号'等）
  3. 路由节点输出 mail_intent（'出入库'/'其他'）、is_inoutbound_mail（布尔值）、fixed_reply_message（预设提示语）到 variables
  4. 后续节点（附件解析、策略校验、简道云写入）启动时先检查 variables 中的 is_inoutbound_mail 和 mail_intent，非出入库邮件立即返回跳过标识
  5. 简道云写入节点根据邮件类型选择回复文案：'其他'邮件发标准拒收提示；校验失败邮件列明错误项；成功写入则发确认通知
  6. 所有节点均将关键中间变量（如 validation_pass、order_form_id、reply_message）存入 variables，最终 result 字段统一输出 human-readable 回复文本
  7. 整个流程支持异常捕获，任何环节报错都返回 error message，不中断主链路
